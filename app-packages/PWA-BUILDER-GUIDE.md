# PWA Builder Guide for Multi Land

This guide provides step-by-step instructions for using PWA Builder to convert Multi Land into native applications for Android and iOS.

## What is PWA Builder?

[PWA Builder](https://www.pwabuilder.com/) is a free tool by Microsoft that helps convert Progressive Web Apps (PWAs) into packages that can be submitted to various app stores, including Google Play Store and Apple App Store.

## Prerequisites

Before using PWA Builder:

1. Your Multi Land app must be deployed to a public HTTPS URL
2. The PWA requirements must be fully implemented:
   - Complete and valid manifest.json
   - Service worker for offline functionality
   - Responsive design that works on mobile devices
   - All icons in both standard and maskable formats

## Step 1: Test Your PWA

1. Visit [PWA Builder](https://www.pwabuilder.com/)
2. Enter your deployed app URL
3. Review the report and address any issues it identifies

## Step 2: Generate Android Package

1. After the report, click "Build" tab
2. Select "Android" option
3. Configure your Android package:
   - Package ID: `com.multiland.app` (must be unique)
   - App name: `Multi Land`
   - Choose signing options (create a new key or use existing)
   - Select icon (the tool will use your PWA icons)
   - Choose display mode (standalone recommended)
4. Click "Generate" to create the Android app package
5. Download the .aab (Android App Bundle) file for submission to Google Play

## Step 3: Generate iOS Package

1. Return to the "Build" tab
2. Select "iOS" option
3. Configure your iOS package:
   - Bundle ID: `com.multiland.app` (must be unique)
   - App name: `Multi Land`
   - Team ID (from your Apple Developer account)
   - Select your icon (the tool will use your PWA icons)
   - Configure display options
4. Click "Generate" to create the iOS Xcode project
5. Download the generated project

## Step 4: Testing Android Package

1. Use the test .apk file to install on Android devices directly
2. Test all functionality:
   - Login/registration with email
   - Course browsing and enrollment
   - Content viewing (videos, PDFs, text)
   - Payment processing with Stripe
   - Offline functionality
   - Language switching between English, Russian, and Georgian

## Step 5: Finalizing iOS Package

1. Open the downloaded Xcode project
2. Connect your Apple Developer account
3. Update signing configuration
4. Test on iOS simulator or real device
5. Make any necessary adjustments
6. Archive the app for App Store submission

## Step 6: App Store Listings

### Google Play Store

1. Login to [Google Play Console](https://play.google.com/console/)
2. Create a new app
3. Fill in all required details:
   - Title: "Multi Land"
   - Short description: "Multilingual learning platform with courses in English, Russian, and Georgian"
   - Full description (include key features, benefits, and languages supported)
   - Select Education category and appropriate subcategory
   - Upload screenshots captured from testing
   - Upload your feature graphic (1024x500px)
   - Complete content rating questionnaire
   - Set up pricing (free with in-app purchases)
4. Upload the .aab file generated by PWA Builder
5. Complete the store listing and submit for review

### Apple App Store

1. Login to [App Store Connect](https://appstoreconnect.apple.com/)
2. Create a new app
3. Fill in all required details:
   - App name: "Multi Land"
   - Subtitle: "Learn in English, Russian & Georgian"
   - Description (highlight key features, benefits, and languages)
   - Keywords: education, courses, multilingual, learning, etc.
   - Support URL: your support website
   - Marketing URL (optional)
   - Privacy Policy URL
   - Select Education category
   - Upload screenshots for all required device sizes
   - Set up pricing information
4. Use Xcode to upload your build to App Store Connect
5. Complete store listing and submit for review

## Troubleshooting Common Issues

### Android Issues

1. **Missing Features**: If certain features aren't working, check Android manifest permissions
2. **Splash Screen Problems**: Customize splash screen in the TWA configuration
3. **Payment Processing**: Ensure Stripe is properly integrated and test thoroughly

### iOS Issues

1. **WebView Limitations**: Some features may need native implementation
2. **App Rejection**: Common reasons include:
   - Insufficient native features
   - Poor UI/UX in mobile context
   - Payment mechanisms not using Apple's in-app purchase
